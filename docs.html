1. The default API (passwords)

The easiest way to generate a strong phrase is to call the API - just call https://hipstapas.dev/api to have a new password. That's it! You just created a password using the default options (alphabet consists of small, capital, numbers and special charactes, min. length is 16 and max. length is 32 characters). Options:

  • alphabetSmall = true|false
  • alphabetCapital = true|false
  • alphabetNumber = true|false
  • alphabetSpecial = true|false
  • lengthMin = number between 1 and 2048
  • lengthMax = number between 1 and 2048
  • resultsCount = number between 1 and 100
Example to:
  • generate 5 phrases at once
  • where each phrase is exactly 5 characters long
  • using only small characters and numbers
https://hipstapas.dev/api?resultsCount=5&lengthMin=5&lengthMax=5&alphabetSpecial=false&alphabetCapital=false.

2. Supported HTTP methods

Currently all API endpoints are supporting only the HTTP GET method. Accepting HTTP POST is work in progress. Base API URL is https://hipstapas.dev/api.

3. Generation options

Because there are only GET methods, all options should be set using query parameters. Currently every endpoint supports a resultsCount query parameter, that controls how many items are returned at once. Example: https://hipstapas.dev/api/?resultsCount=5 generates 5 phrases at once and returns an array containing the newly generated strings. For query parameters of type boolean that are not explicitely specified, true is always the default option.

4. Generating UUID

Call https://hipstapas.dev/api/uuid to have a new uuid. That's it!

5. EFF wordlist

Call https://hipstapas.dev/api/wordlist to create a passphrase from 6 words (as described and suggested by EFF https://www.eff.org/de/dice). This is the default option. The count of the generated words can be set between 1 and 50 using the query parameter words, e.g. https://hipstapas.dev/api/wordlist/?words=3.
